¿Cuál es la diferencia entre Internet y la World Wide Web? Internet es una gran red de computadoras alrededor de todo el mundo conectadas entre sí. Por el contrario, la web (la World Wide Web) es una enorme colección de páginas que se asienta sobreesa red de computadoras. (Redacción, 2019)
¿Cuáles son las partes de un URL? Una URL para HTTP (o HTTPS ) suele estar formada por tres o cuatro componentes. Un esquema que identifica el protocolo, un nombre de host que identifica quien contiene el recurso, un camino que identifica la ruta que el usuario quiere acceder y una cadena de consulta. (CICS Transaction Server for Z/OS 6.x, 2025)
¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? Los métodos HTTP ayudan a crear, leer, actualizar y suprimir cosas en objetos de Rule Execution Server. (IBM Operational Decision Manager 8.12.X, 2024)
¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Se debe utilizar el metodo POST ya que los datos no se muestran. (Equipo editorial de IONOS, 2020)
¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Se utiliza el metodo GET para recibir los datos necesarios para acceder a la URL. (Kinsta, 2022)
Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Un código 200 es una respuesta de estado satisfactorio que indica que la solicitud tuvo éxito. (200 OK - HTTP | MDN, 2023)
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Si, es responsabilidad del desarrollador dado que pudo haver sido que alguien eliminó el contenido o lo cambió a otra URL sin pensar en redirigirlo. (Vadavo, 2024)
¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? No del todo, dado que pudo haber sido por un error en los permisos de acceso. Asimismo, existen situaciones en las que el desarrollador tiene responsabilidad como en timeout o en el codigo incorrecto. (Equipo editorial de IONOS, 2019)
¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Que un atributo esté despreciado o desaprobado significa que ya no se recomienda utilizarlas y que incluso en un futuro se eliminen. Algunos ejemplos son los siguientes: acronym, applet, basefont, big, center. (Marco, n.d.)
¿Cuáles son las diferencias principales entre HTML 4 y HTML5? HTML5 se distingue porque ofrece soporte nativo para audio, video y gráficos vectoriales sin plugins, uso libre de MathML y SVG, creación de formas, almacenamiento temporal con bases de datos SQL en lugar de cookies, ejecución de JavaScript y la interfaz en hilos separados, declaraciones DOCTYPE y de codificación más cortas con UTF-8, y reglas de análisis mejoradas sin depender del Lenguaje de Marcado Generalizado Estándar. (Boada & Boada, 2025)
¿Qué componentes de estructura y estilo tiene una tabla? Estructura
table: Elemento que define la tabla.
thead: Encabezado de la tabla. Contiene las filas de encabezado.
tbody: Cuerpo principal de la tabla. Contiene las filas de datos.
tfoot: Pie de la tabla. Puede contener información adicional o resúmenes.
tr: Define una fila dentro de la tabla.
th: Define una celda de encabezado dentro de una fila.
td: Define una celda de datos dentro de una fila.
caption: Título o leyenda de la tabla.
Estilo en CSS
border: Define el borde de la tabla y sus celdas.
background-color: Define el color de fondo de la tabla, filas o celdas.
font-family: Define el tipo de letra del texto en la tabla.
text-align: Define la alineación del texto dentro de las celdas.
padding: Define el espacio entre el texto y el borde de las celdas.
(Tabla - HTML: Lenguaje De Etiquetas De Hipertexto | MDN, n.d.)

¿Cuál es el ciclo de vida de los sistemas de información? Es un sistema, automatizado o manual, que engloba a personas, máquinas y/o métodos organizados para recopilar, procesar, transmitir datos que representan información. Un sistema de información engloba la infraestructura, la organización, el personal y todos los componentes necesarios para la recopilación, procesamiento, almacenamiento, transmisión, visualización, diseminación y organización de la información. (Guerrero, 2015)
¿Cuál es el ciclo de desarrollo de sistemas de información? El ciclo de vida del desarrollo de los sistemas (SDLC, Systems Development Life Cycle) es un enfoque por fases para el análisis y el diseño, cuya premisa principal consiste en que los sistemas se desarrollan mejor utilizando un ciclo específico de actividades del analista y el usuario. (Monterde, n.d.)
